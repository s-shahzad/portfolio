Detection Engineering / Rule Authoring

Detection Library

Production-style rules authored alongside the Universal NIDS case study. Each rule maps to one of the four hybrid-detection engines used in that project — signature, anomaly, supervised, unsupervised — and to the fusion stage that consolidates agreeing signals before an analyst is paged.

Signature
Deterministic IOC / pattern match. Sigma, Suricata.
Anomaly
Statistical drift from baseline. SPL stddev, KQL series.
Supervised
CatBoost classifier on labeled flows.
Unsupervised
IsolationForest scoring on encoded flow features.
Fusion
Agreement count ≥ 2 across engines. Paging-grade.

Methodology

How these connect to the NIDS run

The NIDS case study reports an engine_distribution across signature, supervised, unsupervised, and fusion engines. Every detection here is shaped to feed exactly one of those engines. A fusion event in the live dashboard means at least two of these rule families fired on the same entity inside the agreement window — that’s the bar at which an analyst is paged. The fusion threshold is documented in the bundle metadata (fusion_min_agreement_count).

The full library, including this README and tuning notes, is on GitHub.
